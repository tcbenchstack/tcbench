    About

    This website documents code and data artifacts related to the IMC23 submission #132 titled

    Contrastive Learning and Data Augmentation in Traffic Classification via a Flowpic Representation Replicating and Reproducing ‚ÄúA Few Shots Traffic Classification with mini-FlowPic Augmentations‚Äù from IMC‚Äô22

    Our submission investigates the role of data augmentation by using both supervised and contrastive learning techniques across 4 datasets.

    It replicates and reproduces the following paper from the IMC22 program

    We adopt the same traffic representation used in imc22-paper, namely a Flowpic -- a summarization of the packet size time series of a flow by means of frequency histograms extracted from consecutive time windows of the flow -- applied on the ucdavis-icdm19.

    In the first part of the submission we investigate how augmentations affect classification performance -- the study considers 3 image transformations (rotation, color jitter, horizontal flip) and 3 time series transformations (time shift, packet drop, change rtt) applied to packets timestamps -- when used either in a fully supervised setting or via contrastive learning.

    Key takeaways from reproducibility

    1. We can only partially reproduce the results from imc22-paper on ucdavis-icdm19. Specifically, we uncover a data shift present in the dataset itself which justifies our results; yet, we cannot comment on why this was not detected in imc22-paper.

    2. Simply based on the ucdavis-icdm19 dataset, and differently from the argumentation presented in imc22-paper, we do not find statistical significance differences across the different augmentations.

    3. Contrastive learning can help to "bootstrap" a model in an unsupervised fashion, yet relying on more samples is beneficial to boost performance.

    Then, in the second part of the submission we replicate the analysis testing the same 6 augmentations across 3 other datasets.

    Key takeaways from replicability

    Using multiple datasets allow to confirm the argument of the imc22-paper, i.e., Change RTT augmentation used in ucdavis-icdm19 is superior to the alternative transformations presented in the paper.
